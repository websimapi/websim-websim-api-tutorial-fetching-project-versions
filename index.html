WebSim Version Logic

A technical guide on programmatically identifying the latest version of any WebSim project.

1. Project Detection

The application identifies the current project environment through the following sequence:

  1. Initial Identification: The app calls window.websim.getCurrentProject(). This returns a summary object containing the ID and the current active version.
  2. Detailed Metadata: To get specific version stats and bookmarks, it performs a fetch to /api/v1/projects/${id}.
javascript 
const projectSummary = await window.websim.getCurrentProject();
const response = await fetch(`/api/v1/projects/${projectSummary.id}`);
const { project } = await response.json();

// Prefer the version from the context (projectSummary) to know 
// exactly which revision is loaded, rather than the latest (project.current_version).
const activeVersion = projectSummary.version || project.current_version;

console.log("Current ID:", project.id);
console.log("Active Version:", activeVersion);

2. Version Detection & Branching Logic

The version history isn't just a linear list; it's a reconstructed tree built via the ancestry path.

A version is "Off-Branch" if it's not in the direct parent chain of the current bookmark.
javascript 
async function getAncestryPath(projectId, currentVersion) {
    const res = await fetch(`/api/v1/projects/${projectId}/revisions`);
    const { revisions } = await res.json();
    
    const parentMap = new Map();
    revisions.data.forEach(rev => {
        parentMap.set(rev.project_revision.version, rev.project_revision.parent_revision_version);
    });

    const mainBranch = new Set();
    let cursor = currentVersion;
    while (cursor) {
        mainBranch.add(cursor);
        cursor = parentMap.get(cursor);
    }
    return mainBranch;
}

3. Real-time Synchronization

Because project state can change while the UI is idle, implement a refresh mechanism using custom events or visibility listeners.

javascript 
window.addEventListener('focus', async () => {
    // Re-verify project state when user returns to tab
    const project = await window.websim.getCurrentProject();
    updateUI(project);
});
